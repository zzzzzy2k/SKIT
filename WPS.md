## 项目命名建议

| 中文名       | 英文名             | 含义                                                         |
| :----------- | :----------------- | :----------------------------------------------------------- |
| **食刻**     | **SKit**           | “食” + “时刻”，谐音“时刻”，也暗含“厨房一刻”。Kit 取自 Kitchen。 |
| **知食分子** | **KnowWhatToCook** | 直白易懂，偏趣味。                                           |
| **随意小厨** | **PickADish**      | 强调随机挑选的轻松感。                                       |
| **食材巧配** | **MatchIng**       | Match + Ingredient，突出食材匹配。                           |

**推荐组合**：中文 **「食刻」**，英文 **「SKit」**（简洁好记，域名资源多）。
备选：中文 **「知食分子」**，英文 **「KnowWhatToCook」**（更直白但较长）。

## 开发顺序建议

1. 先完成 **阶段0 + 阶段1**，产出可用的 `recipes.json`。
2. 再实现 **阶段2** 的核心页面，建议顺序：首页（食材筛选+列表）→ 详情页（含缩放、购物清单）→ 收藏夹页。随机推荐可以最后补。
3. 完成核心功能后，一次性做 **阶段3** 的响应式优化。
4. 最后测试部署。

这样你可以在第3天左右就有一个能跑通核心流程的 Demo（虽然没有样式美化，但能用）。
