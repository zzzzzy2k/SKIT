# 产品需求文档（PRD）- 做饭决策助手

**版本**：v1.0
**更新日期**：2026-05-01
**产品形态**：响应式网页（PC/手机）
**目标用户**：独居或小户型的打工人、学生，自己做饭但每天纠结“吃什么”

------

## 1. 项目背景与目标

### 1.1 背景

用户每天面临“自己做饭，吃什么”的决策疲劳，缺乏菜谱灵感，且现有菜谱应用往往内容繁杂、不适合一人食或缺少食材筛选功能。

### 1.2 核心目标

- 帮助用户快速找到一个**可做、想做的菜**，减少决策时间。
- 支持按**已有食材**筛选菜谱，减少食材浪费。
- 提供**详细做法**和**人数自动调整**功能，适配一人食或多人场景。
- 允许用户**收藏**喜欢的菜谱，后续可生成**购物清单**。

### 1.3 范围（v1.0）

- **包含**：随机推荐、食材筛选、菜谱详情（做法+人数调整）、收藏/取消收藏、个人购物清单生成（基于当前菜谱）。
- **不包含（计划后续版本）** ：全局热度排行榜（需要后端和数据库）、用户系统、菜谱数据自动更新、社区评论、智能推荐算法。

------

## 2. 用户角色与场景

| 角色           | 典型场景                                                     |
| :------------- | :----------------------------------------------------------- |
| **独居打工人** | 下班后想快速做一顿饭，不知道吃什么，希望随机推荐一道能利用冰箱里现有食材的菜。 |
| **学生**       | 预算有限，希望用仅有的鸡蛋、番茄等基础食材做出可口的菜。     |
| **二人食家庭** | 需要将菜谱分量从一人份调整为两人份，并生成购物清单去采购。   |

------

## 3. 功能需求（按优先级）

### P0（必须有，MVP核心）

#### F1 随机推荐一道菜

- **描述**：用户点击“随机推荐”按钮，系统从当前筛选条件下的菜谱中随机选择一道菜，并跳转至详情页。
- **规则**：如果未进行任何食材筛选，则从全部菜谱中随机；如果已筛选食材，则仅从符合条件的菜谱中随机。
- **交互**：按钮明显，点击后立即响应。

#### F2 按食材筛选菜谱

- **描述**：用户可以通过勾选**具体食材**（如“鸡蛋”“番茄”“土豆”等）来筛选菜谱。系统只显示包含所有已选食材的菜谱。
- **食材来源**：从HowToCook菜谱数据中提取所有食材的去重列表（静态数据）。
- **界面**：食材以复选框或标签形式排列，支持搜索或折叠（如果食材过多）。
- **显示结果**：列出符合条件的菜谱名称，点击可进入详情页。

#### F3 菜谱详情页（含详细做法）

- **内容**：
  - 菜名
  - 必备食材和工具（带原始用量）
  - 详细步骤（从挑选食材开始，按顺序）
  - 难度星级（从HowToCook的★符号解析）
  - 预计耗时（如有）
- **数据来源**：HowToCook仓库中的Markdown文件内容（预先抓取为静态JSON）。

#### F4 人数调整（食材用量缩放）

- **描述**：在菜谱详情页提供一个数字输入控件（默认值为1），支持用户输入人数（正整数，限制1~10）。
- **行为**：根据原始一人份的食材用量，等比例缩放所有食材的数量并重新显示。
- **注意**：仅影响展示，不修改原始数据。

#### F5 收藏/取消收藏菜谱

- **描述**：用户在详情页可以点击“收藏”按钮，将该菜谱保存到个人收藏夹。
- **存储**：使用浏览器的 localStorage，无需登录。
- **收藏夹页面**：列出所有已收藏的菜谱，支持取消收藏。

#### F6 生成购物清单（基于当前菜谱）

- **描述**：在菜谱详情页，提供一个“生成购物清单”按钮。点击后，根据当前人数调整后的食材用量，生成一份纯文本清单（例如：“鸡蛋 x3个，番茄 x2个”）。
- **交互**：可复制到剪贴板或直接显示在模态框中。

#### F7 响应式布局

- **要求**：网页在手机（宽度<640px）、平板、桌面（宽度>1024px）上均能正常浏览和操作，布局自适应。

### P1（重要但可后续迭代）

#### F8 热度排行榜（全局版）

- **描述**：展示“最多人收藏”的菜谱排行（需要后端聚合所有用户的收藏数据）。
- **计划**：v2.0实现，v1.0不做。

#### F9 菜谱数据自动更新

- **描述**：当HowToCook仓库更新时，应用能自动或手动触发重新抓取最新菜谱。
- **计划**：v2.0实现（需要后端任务调度），v1.0使用固定的静态数据。

### P2（锦上添花）

#### F10 我的历史浏览记录

- 记录用户最近看过的菜谱（最多20条），方便回溯。

#### F11 搜索菜谱（按菜名）

- 提供搜索框，模糊匹配菜名。

## 4. 非功能需求

| 类型         | 要求                                                         |
| :----------- | :----------------------------------------------------------- |
| **性能**     | 首屏加载时间 ≤ 2秒（在4G网络下）；菜谱详情页切换无延迟。     |
| **可用性**   | 页面交互直观，无需用户培训；食材筛选支持多选且即时刷新结果。 |
| **兼容性**   | 支持最近2个版本的Chrome、Safari、Firefox；手机端iOS 14+，Android 8+。 |
| **离线可用** | 除首次加载数据外，后续使用不需要网络（因为数据全在本地JSON）。 |
| **可维护性** | 代码结构清晰，数据与视图分离，将来替换数据源或增加后端时改动成本低。 |
| **部署**     | 可托管在静态网站服务（如Vercel、Netlify），无需自建服务器。  |

## 5. 数据需求

### 5.1 数据来源

- **主数据**：GitHub仓库 [Anduin2017/HowToCook](https://github.com/Anduin2017/HowToCook) 中的所有菜谱Markdown文件。
- **数据抓取频率**：v1.0仅做一次静态抓取，不自动更新。

### 5.2 数据结构（需从Markdown解析的字段）

| 字段                        | 类型   | 说明                                           |
| :-------------------------- | :----- | :--------------------------------------------- |
| `id`                        | string | 文件名（如“西红柿炒鸡蛋”）作为唯一标识         |
| `title`                     | string | 菜名（从第一个#标题提取）                      |
| `ingredients`               | array  | 食材名称列表（去重，纯文本）                   |
| `ingredients_with_quantity` | array  | 原始食材行，保留数量和单位（如“鸡蛋 3个”）     |
| `steps`                     | array  | 步骤列表（从“## 操作步骤”下提取）              |
| `tools`                     | array  | 工具列表（从“必备原料和工具”中区分原料与工具） |
| `difficulty`                | enum   | 难度星级（0~5颗星，从正文中的★数量解析）       |
| `raw_markdown`              | string | 原始Markdown内容（备用）                       |

### 5.3 本地存储设计（localStorage）

| 键名        | 类型          | 说明                                     |
| :---------- | :------------ | :--------------------------------------- |
| `favorites` | array[string] | 收藏的菜谱id列表                         |
| `history`   | array[string] | 最近浏览的菜谱id列表（最多20，先进先出） |

## 6. 约束与假设

### 6.1 技术约束

- 前端必须为纯静态（HTML/CSS/JS + React/Vite 打包产物），不允许运行时依赖自建后端。
- 数据必须预先打包到发布文件中，不可在运行时从GitHub API实时拉取（避免跨域、限速、不稳定）。

### 6.2 业务假设

- 用户使用现代浏览器，支持localStorage和ES6。
- 用户对“人数调整”的理解是基于一人份菜谱的线性缩放（某些调味料可能不需要严格线性，但初版接受线性假设）。
- 用户默认接受一个人做饭的场景，因此一人份为基准。

### 6.3 法律合规

- HowToCook 仓库采用 MIT 许可证，可自由使用和修改，需保留署名（在页面底部注明“菜谱数据来自 HowToCook 开源项目”）。

## 7. 用户界面草图（文字描述）

### 7.1 首页

- 顶部大标题“今天吃什么？”
- 一个醒目的 **随机推荐** 按钮。
- 下方为 **食材筛选区**（多选框组，两列或横向滚动）。
- 筛选后展示 **菜谱列表**（每个菜名可点击）。
- 底部导航栏：首页 | 收藏夹 | 关于/设置（可选）。

### 7.2 菜谱详情页

- 菜名、难度星级（★图示）。
- 人数选择器（带 +/- 按钮和直接输入）。
- 食材清单（按缩放后显示）。
- 工具清单。
- 操作步骤（有序列表）。
- 按钮：生成购物清单 | 收藏/已收藏。
- 底部返回按钮。

### 7.3 收藏夹页

- 列表展示所有收藏的菜谱（显示菜名、难度）。
- 每个条目带取消收藏的图标。
- 空状态提示“暂无收藏，去首页探索吧”。

## 8. 验收标准

| 功能     | 验收条件                                                     |
| :------- | :----------------------------------------------------------- |
| 随机推荐 | 每次点击推荐不同的菜谱（在筛选条件内）。                     |
| 食材筛选 | 勾选“鸡蛋”和“番茄”，只显示同时包含这两样食材的菜谱。         |
| 人数调整 | 将人数从1改为2，食材数量翻倍（如果原食谱写“鸡蛋 3个”，应显示“鸡蛋 6个”）。 |
| 收藏     | 收藏后刷新页面，收藏状态保留；收藏夹页能看到。               |
| 购物清单 | 生成的清单文本包含所有食材的名称和缩放后的用量，格式清晰易读。 |
| 响应式   | 在iPhone SE（375px）上不出现横向滚动，按钮可点击。           |

## 9. 后续版本规划（v2.0）

- 全局热度排行榜（基于真实收藏数，需要后端）。
- 用户账号系统（登录后收藏跨设备同步）。
- 自动同步HowToCook仓库更新。
- 食材过敏/忌口过滤。
- 按烹饪时间、难度筛选。
- 菜谱分享功能（生成图片或链接）。
