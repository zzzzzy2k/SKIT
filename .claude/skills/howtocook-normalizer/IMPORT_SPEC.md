# HowToCook 菜谱导入规范

本文档定义了 HowToCook 菜谱 Markdown 文件的导入规范。不符合规范的文件将无法被正确解析或产生错误数据。

## 必需的文档结构

每个菜谱 `.md` 文件必须包含以下 H2 标题的 section（按顺序）：

```markdown
# 菜名的做法

## 必备原料和工具
- 食材1
- 食材2

## 计算
- 食材1 500 g
- 食材2 2 勺

## 操作
- 步骤1
- 步骤2

## 附加内容
- 提示1
```

### 标题关键字匹配

| Section | 匹配关键字 |
|---------|-----------|
| 食材 | `必备原料和工具`、`原材料`、`原料`、`食材`、`配料` |
| 计算 | `计算` |
| 步骤 | `操作步骤`、`操作`、`步骤`、`制作` |
| 提示 | `附加内容`、`附加`、`提示`、`小贴士`、`注意事项` |

## 食材格式规则

### 一行一个食材

每行只放一个食材，不要用顿号合并：

```
<!-- 正确 -->
- 葱
- 姜
- 蒜

<!-- 错误 -->
- 葱、姜、蒜
```

### 数量格式

数量在食材名后面，用空格分隔：

```
<!-- 正确 -->
- 五花肉 500 g
- 盐 5 g

<!-- 错误 -->
- 五花肉500g
- 500g五花肉
```

### 无精确数量

用"适量"、"少许"等关键词：

```
- 盐 适量
- 胡椒粉 少许
```

### 二选一食材

用 `or`、`或者`、`或` 表示二选一（系统会取第一个）：

```
<!-- 正确：系统会取"白糖" -->
- 白糖 or 冰糖

<!-- 正确：无空格也支持 -->
- 朗姆酒或啤酒

<!-- 正确：括号内的 or 不会被分割 -->
- 鲜面条（韭叶 or 二细均可）
```

## 计算 section 规则

`## 计算` section 中的行格式与食材 section 相同，但提供精确数量。系统会优先使用计算 section 中的数量。

```
## 计算
- 五花肉 500 g
- 葱 2 根
- 姜 3 片
```

顿号合并的计算行也会被自动拆分：
```
<!-- 会被拆分为 4 个独立条目 -->
- 盐 2 g、生抽 15 ml、老抽 10 ml、蚝油 10 ml
```

## 工具识别

包含以下关键词的行会被识别为工具（不作为食材）：

`锅`、`刀`、`砧板`、`碗`、`盘`、`勺`、`铲`、`盆`、`烤箱`、`微波炉`

## 注意事项

### 常见问题

1. **顿号合并食材**：`葱、姜、蒜` 会被自动拆分为 3 个食材，但建议在源文件中一行一个
2. **or 替代**：`白糖 or 冰糖` 会取第一个 `白糖`，括号内的 `or` 不受影响
3. **backtick 格式**：`` `大肉`、`鸡蛋` `` 中的 backtick 会被去掉
4. **分类前缀**：`主料：`、`辅料：`、`炒料：` 等前缀会被去掉
5. **H1 标题**：必须存在 `# 菜名的做法` 格式的标题

### 不会处理的情况

- `## 操作` 中的食材引用（这是正常的步骤描述）
- 括号内的可选说明：`鸡蛋（可选）` 保留原样
- 图片和链接：不影响解析

### 自动修复

导入时系统会自动处理：
- 去掉 backtick：`` `大肉` `` → `大肉`
- 去掉分类前缀：`主料：大肉` → `大肉`
- 拆分顿号：`葱、姜` → `葱` + `姜`
- 拆分顿号（计算 section）：`盐 2 g、生抽 15 ml` → `盐 2 g` + `生抽 15 ml`
- 处理 or/或者/或：`白糖 or 冰糖` → `白糖`（括号内的 or 不受影响）
- 归一化单位：`克` → `g`，`毫升` → `ml`
- 清理单位注释：`g（一小块）` → `g`
- 无空格数量：`生抽15ml` → `生抽 15 ml`
